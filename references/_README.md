## 命名规范（v2 起作为权威源）

```
{学段}_{课题}_{主情境名}_v{版本号}_{YYYYMMDD}
```

- **学段**：小学 / 初中 / 高中（必要时加水平段，如"小学水平二"）
- **课题**：教材上的标准动作名（如"蹲踞式起跑""双手胸前传接球"）
- **主情境名**：全稿核心情境词（如"速度猎豹""篮球指挥官"）
- **版本号**：v1 / v2 / v3（重大重构升版本，小修小补不改版本）
- **日期**：YYYYMMDD 8 位

**【v2/v3 字段双源统一】** 母版元信息里的"学段/课题/主情境名"等字段**必须与本命名规范完全一致**。**本 README 为权威源**。

## 生成流程（agent 接到新课题必走 · v3 强化版 · 6 项必确认）

```
1. 读 01_模板/风格特征_生成校验清单_v3.md       ← 5 特征 + 9 硬约束 + 特征冲突解决表
2. 读 01_模板/模板_试讲逐字稿_主情境串课_v3.md  ← 8 大模块 + 占位符
3. 读 02_示范稿/中最近的 1 篇                  ← 风格体感参考
4. 与用户确认 6 项：
   - 学段（小学/初中/高中 + 水平段）
   - 课题
   - 教材版本
   - 主情境方向
   - 【v2/v3】课题边界 5 项（空间/空间属性/器材/人数/负荷）
   - 【v2/v3】课题类型（5 类冲突解决表对应）
5. 按母版生成，逐项对照校验清单打勾（19 条自检）
6. 对未达标项：要么回退方案写明，要么显式标注"有意省略"
7. 交付前必须输出"自检报告"
```
